Pourquoi git ?

Git est un outil de gestion de version qui nous a été enseigné lors de ma formation. Il est extrêmement utile pour suivre et gérer les modifications apportées à un projet logiciel au fil du temps.

Une des fonctionnalités les plus puissantes de Git est son système de branches. Les branches permettent de travailler sur différentes versions d'un projet en parallèle, sans interférer les unes avec les autres. Par exemple, nous pouvons créer une branche pour travailler sur une nouvelle fonctionnalité, tandis que la branche principale (généralement appelée 'master' ou 'main') reste stable.

Cela permet une meilleure organisation du travail en équipe, car chaque membre peut travailler sur sa propre branche sans perturber le travail des autres. Ensuite, cela facilite la gestion des fonctionnalités en cours de développement, car chaque branche peut être fusionnée avec la branche principale une fois que le travail est terminé et testé.

Les branches sont très utiles pour expérimenter de nouvelles idées ou résoudre des problèmes sans risquer de casser le code principal. Si une nouvelle fonctionnalité ne fonctionne pas comme prévu, il est facile de revenir à la branche principale sans perdre de travail.

Pour notre projet Slime GO nous avons utilisé gitlab qui nous a permit en partie de se répartir le travaille de s'organiser et de faire des sauvegardes et différentes versions du projet. Nous avons appris a utilisé git tout au long de notre formation pour les projets en groupes ou à plusieurs et c'est un outil intéressant puisqu'il est très utile. Depuis j'utilise git pour la plus part de mes projets.
