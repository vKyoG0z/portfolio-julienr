ExpressJS

Durant ma formation, nous avons appris le fonctionnement de Express.js, un framework web pour Node.js. Son système de routage est un élément central qui permet de gérer efficacement les requêtes HTTP et de diriger les utilisateurs vers les différentes pages de notre application.

Actuellement vous êtes sur la page expressjs.html qui est définis sur la route /formationdashboard/expressjs comme le montre l'image ci-dessus. Vous pouvez également vérifier dans votre url !

Tout commence par l'utilisation de la méthode app.get, qui écoute les requêtes GET sur une URL spécifique. Par exemple, si un utilisateur entre l'URL '/accueil' dans son navigateur, Express.js intercepte cette requête et exécute la fonction associée. Mais Express.js ne se limite pas aux simples requêtes GET. Avec les méthodes app.post, app.put, app.delete, etc. Nous pouvons également gérer d'autres types de requêtes HTTP, comme les soumissions de formulaires, les mises à jour de ressources, ou les suppressions de données qui sont des choses beaucoup utilisées dans le monde du web.

L'un des avantages de Express.js est sa flexibilité. En combinant différents types de routes nous pouvons créer des applications web complexes avec une logique de navigation fluide.

Express.js nous a ouvert les portes d'un monde de développement web dynamique et modulable grâce à son système de routage puissant et sa simplicité d'utilisation, nous avons pu créer des applications web (comme ce portfolio sur lequel vous vous trouvez) tout en gardant notre code bien organisé et facile à maintenir."
